# 04 — Purchase Orders UI Specification

**Source:** Crunchwork Pulse vendor portal — staging-iag (observed 2026-06-07)
**Route:** Hamburger menu → Purchase Orders
**CW API reference:** Insurance REST API v17 §3.3.7 (Purchase Order)

---

## 1. Purchase Orders List Page

### 1.1 URL
`https://staging-iag.crunchwork.com/pulse/vendor/purchase-orders` (via hamburger menu)

### 1.2 Layout

Follows the same consistent pattern as Quotes:

**Left filter sidebar — "Search Purchase Orders":**
| Filter | Type |
|--------|------|
| Title/Name | Text input |
| PO Number | Text input |
| Status | Dropdown |
| Total To | Currency input |
| Total From | Currency input |
| PO Type | Dropdown |
| From Date | Date picker (dd/mm/yyyy) |
| To Date | Date picker (dd/mm/yyyy) |
| CLEAR link | Resets all |

**Main content area:**
| Element | Details |
|---------|---------|
| Heading | "All Items" or "Search Results" |
| Total count | "Total Items found: N" |

### 1.3 Table Columns (expected based on consistent patterns)

| Column | Sortable | Description |
|--------|----------|-------------|
| Date | ↕ Yes | PO issue date |
| PO No | ↕ Yes | Purchase order number |
| Title | ↕ Yes | PO title/name |
| Total | ↕ Yes | PO total amount |
| Status | No | Status badge |
| Type | No | PO type |
| Actions | No | VIEW button (eye icon) |

### 1.4 Status Values (expected)

| Status | Colour | Description |
|--------|--------|-------------|
| Issued | Blue | PO sent to vendor |
| Accepted | Green | Vendor confirmed |
| In Progress | Yellow | Work underway |
| Completed | Green | Work finished |
| Cancelled | Red | Voided |

## 2. Purchase Order Detail Page

### 2.1 URL Pattern
`/pulse/vendor/purchase-orders/{po-id}` (inferred)

### 2.2 Expected Layout

Based on the consistent detail page pattern:

**Header:**
- Organisation name | PO Type | PO Number | Address
- Status badge
- Action buttons (TEAMS, Save, etc.)

**Tabs (expected):**
| Tab | Content |
|-----|---------|
| OVERVIEW | PO details, parties, financial |
| ACTIVITIES | Tasks and Appointments |
| COMMUNICATIONS | Emails |
| ATTACHMENTS | Files |
| TIMELINE | Notes and audit |

### 2.3 Overview Content (expected)

**PO Identifiers:**
| Field | Description |
|-------|-------------|
| PO Number | System reference |
| External ID | CW UUID |
| Name | PO title |
| Status | Current status |
| Type | PO type (Standard, Variation) |
| Vendor | Allocated vendor |

**Service Window:**
| Field | Description |
|-------|-------------|
| Start Date | Work start |
| End Date | Work end |
| Start Time | Daily start |
| End Time | Daily end |

**Financial:**
| Field | Description |
|-------|-------------|
| Total | Original PO amount |

**Parties (same pattern as Quotes):**
- PO To (vendor/recipient)
- PO For (insured/customer)
- PO From (issuing insurer)

**Line Items:**
Same group → combo → item hierarchy as quotes.

## 3. Create Purchase Order Flow

### 3.1 Context

POs are typically auto-generated by the insurer when a quote is approved. In the contractor chain model, a prime contractor can also issue POs to sub-contractors.

### 3.2 Expected Creation (Sub-contractor PO)

| Field | Type | Required |
|-------|------|----------|
| Job | Select | Yes |
| Source Quote | Select (from approved) | Yes |
| Vendor (sub) | Select | Yes |
| PO Type | Dropdown | Yes |
| Start Date | Date picker | Yes |
| End Date | Date picker | Yes |
| Note | Textarea | No |

## 4. PO Status Flow

```
Issued → Accepted → In Progress → Completed
                  → Rejected
       → Expired
       → Cancelled
```

| From | To | Actor |
|------|----|-------|
| (quote approved) | Issued | Insurer (auto) |
| Issued | Accepted | Vendor |
| Issued | Rejected | Vendor |
| Accepted | In Progress | Vendor |
| In Progress | Completed | Vendor |
| — | Cancelled | Insurer |
| — | Expired | System (time-based) |

---

## 5. Key Observations

- POs are accessed via the hamburger menu (not top navigation)
- Structure mirrors the Quotes list view exactly
- POs are usually received (from insurer) rather than created by the vendor
- Line items on a PO come from the approved quote
- The vendor's primary action is to accept/reject and progress the PO status
- Invoice creation is linked to PO completion (see Invoices spec)
